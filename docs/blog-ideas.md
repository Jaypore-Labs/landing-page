# Blog idea pipeline — Jaypore Labs

Topic focus: **AI · AI products · AI tools · using them in engineering and development.**
Every idea uses an "elephant on a bicycle" pairing — a familiar technical concept (the bicycle) and an unusual frame (the elephant). Reader leans in because the combination surprises them; stays because the pairing actually illuminates something.

Use `/daily-article` skill to turn any row below into a full content package (blog + IG carousel + LinkedIn + X thread). See `.claude/skills/daily-article.md` for the flow.

## How to pick

- **Rank by what our team has shipped** — a playbook you've actually run is worth more than an abstract piece.
- **Rotate weight** — one AI-specific topic (RAG, MCP, agents), one workflow topic (evals, observability, cost), one craft topic (prompts, docs, code review) per three-post cycle.
- **Avoid pairings that need 3 paragraphs to explain.** If the elephant takes work to mount the bicycle, skip it.
- **Ground in a real person.** Every article that names a practitioner reads ten times stronger than one that doesn't. "The psychiatrist we shipped to…" beats "users."

## Rules that make this work

1. **One pairing per article.** Don't mix two metaphors.
2. **State the pairing in the first two sentences.** No long warm-up.
3. **Each lesson should be carousel-ready** — <40 chars, standalone.
4. **The CTA is "if you're working on X, we'd love to hear."** Never a pitch.
5. **Use absolute dates** — `2026-04-20`, not "yesterday".
6. **Voice markers that land on-brand:** "the friendly way", "industry-first, then AI-first", "no buzzword bingo", "real users, not dry-runs", "boring is what scales".
7. **Don't say "we ship AI."** We ship AI-enabled software.
